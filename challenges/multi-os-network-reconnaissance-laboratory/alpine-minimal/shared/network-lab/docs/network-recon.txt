Network Reconnaissance Laboratory Documentation
==============================================

This laboratory environment consists of four distinct Linux distributions:
- Alpine Linux (Minimal) - Primary scanner target
- Ubuntu Server - Web services and database
- CentOS Stream - Mail and file services  
- Debian Stable - DNS and authentication services

Each system is configured with unique service fingerprints for nmap training.

Target Information:
- Alpine: SSH(22), TFTP(69), FTP(28), NetCat(1337)
- Ubuntu: HTTP(80), HTTPS(443), MySQL(3306), SSH(2222)
- CentOS: SMTP(25), POP3(110), IMAP(143), FTP(21)
- Debian: DNS(53), LDAP(389), Kerberos(88), SSH(22)

Scanning Techniques to Practice:
1. TCP SYN Scan (-sS)
2. UDP Scan (-sU) 
3. OS Detection (-O)
4. Service Version Detection (-sV)
5. Script Scanning (-sC)
6. Stealth Scanning (-f, -D)
7. Timing Templates (-T0 to -T5)

Hint: Some services run on non-standard ports for evasion training.
Check the shared resources for additional network configuration details.
