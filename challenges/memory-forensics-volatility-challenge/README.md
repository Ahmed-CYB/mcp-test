# Memory Forensics Challenge

## Learning Objectives
- Understand how to use Volatility for memory analysis
- Identify suspicious processes and extract information

## Instructions
- Access the victim container at http://victim:8080 to download the memory dump
- Use Volatility on the attacker (Kali Linux) to analyze the dump

## Hints
1. Start by listing processes to find anything unusual.
2. Check for hidden processes or DLLs.
3. Look for the flag in suspicious process memory dumps.
